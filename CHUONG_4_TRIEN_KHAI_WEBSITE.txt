CHƯƠNG 4: TRIỂN KHAI WEBSITE

4.1. Triển khai các chức năng cho phân hệ người dùng

Phân hệ người dùng của hệ thống được xây dựng theo mô hình tách biệt giữa giao diện và xử lý nghiệp vụ. Ở phía giao diện, dự án sử dụng ReactJS kết hợp TypeScript, được khởi tạo và build bằng Vite, đồng thời sử dụng thêm các thư viện như React Router DOM để điều hướng trang, Axios để gọi API, TanStack React Query để truy vấn dữ liệu, Ant Design để hỗ trợ một số thành phần giao diện, và Lucide React để hiển thị biểu tượng. Toàn bộ ứng dụng frontend được khởi tạo tại file Frontend/my-react/src/main.tsx, trong đó tích hợp BrowserRouter và QueryClientProvider.

Phân hệ người dùng gồm các chức năng chính như: đăng ký tài khoản, đăng nhập, xem trang chủ, xem danh sách khách sạn, xem chi tiết khách sạn, tìm kiếm dữ liệu dịch vụ, đặt chỗ khách sạn, thanh toán, xem lịch sử đặt chỗ và một số trang tài khoản cá nhân. Frontend giao tiếp với backend thông qua các API dạng REST, với cấu hình gọi API tập trung tại file Frontend/my-react/src/services/api.ts. Backend cung cấp API bằng Node.js + Express, trả dữ liệu JSON, xác thực người dùng bằng JWT, và lưu trữ dữ liệu bằng MySQL thông qua thư viện mysql2/promise.

4.1.1. Chức năng đăng ký tài khoản

a) Phía frontend

- Xây dựng bố cục trang bằng HTML/JSX:

Chức năng đăng ký được triển khai tại file Frontend/my-react/src/pages/DangKy.tsx. Giao diện được xây dựng theo bố cục hai cột, gồm khối biểu mẫu nhập thông tin tài khoản và khối trình bày thông điệp giới thiệu. Form đăng ký gồm các trường: tên đăng nhập, họ tên, email, số điện thoại, mật khẩu và xác nhận mật khẩu. Ngoài ra còn có nút đăng ký và liên kết chuyển sang trang đăng nhập.

- Kỹ thuật định dạng bằng CSS:

Giao diện sử dụng kiểu thiết kế hiện đại với nền gradient, card bo góc, đổ bóng và các vùng nhập liệu dạng khối. CSS được kết hợp giữa inline style trong file component và file dùng chung Frontend/my-react/src/assets/css/DangNhap.css. Form được căn giữa, khoảng cách giữa các trường được bố trí đều, màu sắc theo tông xanh dương nhạt phù hợp với ứng dụng du lịch.

- Sử dụng JavaScript, ReactJS để lập trình chức năng:

Component quản lý state bằng useState cho các trường nhập liệu. Khi người dùng bấm nút đăng ký, hàm handleSubmit() kiểm tra dữ liệu bắt buộc, kiểm tra khớp mật khẩu xác nhận, sau đó gọi hàm registerWithAPI() trong file Frontend/my-react/src/utils/auth.ts. Hàm này gửi request POST đến API /api/auth/register. Nếu đăng ký thành công, hệ thống điều hướng người dùng sang trang đăng nhập và hiển thị thông báo đã đăng ký thành công.

b) Phía backend

- Triển khai lớp tầng DataAccess/Model:

Dữ liệu người dùng được lưu trong bảng Users. Việc thêm người dùng mới được thực hiện trực tiếp trong Backend/my-node-app/services/auth.service.js thông qua truy vấn SQL INSERT INTO Users (...). Trong quá trình này, hệ thống kiểm tra trùng username hoặc email bằng truy vấn SELECT * FROM Users WHERE username = ? OR email = ?.

- Triển khai lớp tầng Business/Service:

Tại file Backend/my-node-app/services/auth.service.js, hàm register() đảm nhiệm nghiệp vụ đăng ký tài khoản. Hệ thống kiểm tra tài khoản đã tồn tại hay chưa, mã hóa mật khẩu bằng bcryptjs, sau đó thêm bản ghi người dùng vào cơ sở dữ liệu với vai trò mặc định là CUSTOMER và trạng thái hoạt động là true.

- Triển khai lớp tầng Controller:

File Backend/my-node-app/controllers/auth.controller.js tiếp nhận request đăng ký thông qua hàm register(). Controller kiểm tra dữ liệu đầu vào cơ bản gồm username, password, ten, sau đó gọi AuthService.register(). Route tương ứng được khai báo tại Backend/my-node-app/routes/auth.routes.js:

- POST /api/auth/register

API này được frontend gọi từ Frontend/my-react/src/utils/auth.ts.

4.1.2. Chức năng đăng nhập

a) Phía frontend

- Xây dựng bố cục trang bằng HTML/JSX:

Trang đăng nhập được triển khai tại Frontend/my-react/src/pages/DangNhap.tsx. Bố cục gồm phần giới thiệu bên trái và phần form đăng nhập bên phải. Form gồm hai trường chính: tên đăng nhập và mật khẩu. Ngoài ra còn có nút hiển thị/ẩn mật khẩu, thông báo lỗi, thông báo đăng ký thành công và liên kết sang trang đăng ký.

- Kỹ thuật định dạng bằng CSS:

Giao diện sử dụng card lớn có bo góc, nền mờ, hiệu ứng bóng và bố cục chia lưới hai cột. Trường nhập liệu được gắn icon minh họa bằng thư viện lucide-react. Màu chủ đạo là cam và xanh, tạo cảm giác hiện đại và thân thiện.

- Sử dụng JavaScript, ReactJS để lập trình chức năng:

Khi người dùng gửi form, hàm handleSubmit() gọi loginWithAPI() trong Frontend/my-react/src/utils/auth.ts. Hàm này gửi request POST đến /api/auth/login. Nếu đăng nhập thành công, token JWT được lưu vào localStorage với key travelhub_token, đồng thời lưu session vào travelhub_session. Sau đó hệ thống điều hướng: nếu vai trò là admin thì chuyển tới /admin/ThongKe, nếu vai trò là customer thì chuyển tới /mua-sam.

Ngoài ra, file Frontend/my-react/src/services/api.ts cấu hình interceptor để tự động gắn token vào header Authorization cho các request sau này, và tự động chuyển về trang đăng nhập nếu API trả lỗi 401.

b) Phía backend

- Triển khai lớp tầng DataAccess/Model:

Việc truy vấn tài khoản khi đăng nhập được thực hiện trong Backend/my-node-app/services/auth.service.js, bằng câu lệnh tìm người dùng theo username từ bảng Users.

- Triển khai lớp tầng Business/Service:

Hàm login() trong Backend/my-node-app/services/auth.service.js đảm nhiệm nghiệp vụ đăng nhập. Hệ thống kiểm tra: tài khoản có tồn tại hay không, tài khoản có bị khóa hay không, mật khẩu có đúng hay không. Mật khẩu được so sánh bằng bcrypt.compare() nếu dữ liệu là dạng hash; ngoài ra có hỗ trợ so sánh plain text cho dữ liệu mẫu. Sau đó hệ thống tạo JWT token bằng jsonwebtoken, với thông tin maUser, username, vaiTro, thời hạn 1 ngày.

- Triển khai lớp tầng Controller:

File Backend/my-node-app/controllers/auth.controller.js tiếp nhận request đăng nhập qua hàm login(), kiểm tra đầu vào rồi gọi AuthService.login(). Route tương ứng:

- POST /api/auth/login

API này được frontend gọi từ Frontend/my-react/src/utils/auth.ts.

Component còn sử dụng CustomerHotelSearchResults để hiển thị kết quả theo điều kiện tìm kiếm.

Trang sử dụng các file CSS CustomerHome.css và CustomerHotel.css, kết hợp bố cục dạng thẻ card, lưới nhiều cột, khu vực tìm kiếm có biểu tượng và hiệu ứng mở rộng. Giao diện phù hợp với web du lịch, hiển thị danh sách khách sạn dưới dạng các khối trực quan.

- Sử dụng JavaScript, ReactJS để lập trình chức năng:

Truy vấn SQL kết hợp các bảng KhachSan, DichVu, NhaCungCap, và truy vấn con trên bảng LoaiPhong để tính giá thấp nhất còn phòng.

- Triển khai lớp tầng Controller:

File Backend/my-node-app/controllers/khachsan.controller.js có hàm publicGetAll() gọi KhachSanModel.getAll(req.query) và trả về kết quả. Route khai báo tại Backend/my-node-app/routes/customer.routes.js:

Ở frontend, API này được gọi tại Frontend/my-react/src/services/hotelService.ts.

4.3.2. Đóng gói ứng dụng

Source code của hệ thống được tổ chức thành hai phần chính là frontend và backend. Mã nguồn có thể được quản lý bằng Git/GitHub nhằm phục vụ theo dõi thay đổi, cộng tác phát triển và triển khai phiên bản. Cấu trúc thư mục tổng quát của dự án gồm:

- Frontend/my-react/: chứa mã nguồn giao diện ReactJS.
- Backend/my-node-app/: chứa mã nguồn API backend Node.js/Express.
- Backend/my-node-app/config/: cấu hình kết nối cơ sở dữ liệu.
- Backend/my-node-app/routes/: định nghĩa các route API.
- Backend/my-node-app/controllers/: xử lý request/response.
- Backend/my-node-app/models/: thao tác dữ liệu với MySQL.
- Backend/my-node-app/middleware/: middleware xác thực và phân quyền.
- Backend/my-node-app/uploads/: thư mục lưu ảnh upload.

Để cài đặt thư viện, phía frontend sử dụng file package.json với các lệnh:

- npm install
- npm run dev
- npm run build

Trong đó lệnh build frontend theo file Frontend/my-react/package.json là:

npm run build

thực chất chạy:

tsc -b && vite build

Phía backend cũng sử dụng package.json riêng. Các lệnh chính là:

npm install
npm start

hoặc:

npm run dev

Theo source code tại Backend/my-node-app/index.js, backend mặc định chạy ở cổng 5000. Frontend giao tiếp với backend thông qua biến môi trường VITE_API_BASE_URL, nếu không khai báo thì mặc định dùng:

http://localhost:5000

Đối với cơ sở dữ liệu, file Backend/my-node-app/config/db.js cho thấy hệ thống dùng MySQL, với các biến cấu hình:

- DB_HOST
- DB_USER
- DB_PASSWORD
- DB_NAME
- DB_PORT
- DB_CONNECTION_LIMIT

Nếu không cấu hình, hệ thống mặc định dùng:

- host: localhost
- user: root
- password: 123456
- database: travel
- port: 3306

4.3.3. Triển khai ứng dụng

a) Yêu cầu phần cứng

Để triển khai và vận hành hệ thống website, cần chuẩn bị cấu hình phần cứng tối thiểu như sau:

- CPU: Intel Core i3 hoặc tương đương trở lên
- RAM: Tối thiểu 8 GB
- Ổ cứng: Tối thiểu 10 GB trống để chứa source code, thư viện và cơ sở dữ liệu
- Kết nối internet: Cần có để tải thư viện, đồng bộ mã nguồn và truy cập hệ thống
- Băng thông mạng: Đủ ổn định để phục vụ việc gọi API giữa frontend và backend, đặc biệt khi triển khai trên máy chủ thực tế

b) Yêu cầu phần mềm

Hệ thống cần các phần mềm sau để phát triển và triển khai:

- Trình duyệt web: Google Chrome, Microsoft Edge hoặc tương đương
- Visual Studio Code: để chỉnh sửa và quản lý source code
- Node.js và npm: để cài đặt thư viện, build frontend và chạy backend
- MySQL: để lưu trữ dữ liệu hệ thống
- Git: để quản lý phiên bản mã nguồn
- Các công cụ khác nếu cần: Postman để kiểm thử API, terminal hoặc PowerShell để chạy lệnh

c) Quy trình triển khai

Quy trình triển khai hệ thống có thể thực hiện theo các bước sau:

Bước 1: Clone source code

git clone <link-repository>

Bước 2: Cài đặt thư viện frontend

cd Frontend/my-react
npm install

Bước 3: Build frontend

npm run build

Nếu cần chạy ở chế độ phát triển:

npm run dev

Bước 4: Cài đặt thư viện backend

cd Backend/my-node-app
npm install

Bước 5: Cấu hình biến môi trường

Tạo hoặc chỉnh sửa file .env cho backend để khai báo các thông số:

- PORT
- DB_HOST
- DB_USER
- DB_PASSWORD
- DB_NAME
- DB_PORT
- JWT_SECRET
- FRONTEND_URL

Phía frontend có thể cấu hình file .env với biến:

- VITE_API_BASE_URL

Bước 6: Khởi chạy backend

npm start

hoặc:

npm run dev

Bước 7: Truy cập website kiểm tra

Sau khi backend chạy thành công tại cổng 5000 và frontend chạy bằng Vite, người dùng có thể mở trình duyệt và truy cập địa chỉ frontend để sử dụng hệ thống. Các request từ giao diện sẽ được gửi đến backend thông qua API REST đã cấu hình.

Kết luận chương

Qua quá trình phân tích source code, có thể thấy hệ thống website được triển khai theo kiến trúc tách biệt frontend và backend. Phần frontend sử dụng ReactJS, TypeScript và Vite để xây dựng giao diện hiện đại, trong khi backend sử dụng Node.js, Express và MySQL để xử lý nghiệp vụ và lưu trữ dữ liệu. Hệ thống đã xây dựng được đầy đủ nhiều chức năng thực tế cho cả phân hệ người dùng và phân hệ quản trị như đăng ký, đăng nhập, xem và tìm kiếm khách sạn, đặt chỗ, quản lý khách sạn, quản lý đơn đặt, thống kê vận hành và nhiều module nội dung khác. Nội dung triển khai trong chương này phản ánh trực tiếp các thành phần, route, controller, model và API có mặt trong source code hiện tại của dự án.
